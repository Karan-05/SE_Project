# Topcoder Identity Service Tests Improvement Part 1

## Problem Summary
The goal of this project is to enhance the testing framework for the Topcoder Identity Service, specifically focusing on the `RoleController` and `UserController`. The existing implementation is in Java using MySQL, and we are transitioning to TypeScript with PostgreSQL. This transition requires comprehensive unit and end-to-end (E2E) tests to ensure that the functionalities remain intact and that the code quality is maintained.

## Source Data and Schemas
### Source Schema (MySQL)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role_id INT,
    FOREIGN KEY (role_id) REFERENCES roles(id)
);

CREATE TABLE roles (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) UNIQUE NOT NULL
);
```

### Destination Schema (PostgreSQL)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role_id INT REFERENCES roles(id)
);

CREATE TABLE roles (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) UNIQUE NOT NULL
);
```

### Invariants
1. **Unique Constraints**: Both `username` in `users` and `name` in `roles` must be unique.
2. **Foreign Key Integrity**: `role_id` in `users` must reference a valid `id` in `roles`.
3. **Non-null Constraints**: Both `username` and `password` in `users` cannot be null.

## Transformation Steps
1. **Data Migration**: Migrate existing data from MySQL to PostgreSQL while ensuring schema integrity.
2. **Code Refactoring**: Rewrite the existing Java code in TypeScript using Prisma for ORM.
3. **Test Implementation**: Write unit and E2E tests for the `RoleController` and `UserController`.

## SQL or Code Snippets
### Pseudo-code for Data Migration
```typescript
async function migrateData() {
    const users = await mysqlClient.query('SELECT * FROM users');
    for (const user of users) {
        await postgresClient.users.create({
            data: {
                username: user.username,
                password: user.password,
                role_id: user.role_id
            }
        });
    }
}
```

### Pseudo-code for E2E Test
```typescript
describe('UserController E2E Tests', () => {
    it('should create a new user', async () => {
        const response = await request(app)
            .post('/api/users')
            .send({ username: 'testuser', password: 'password123', role_id: 1 });
        expect(response.status).toBe(201);
        expect(response.body.username).toBe('testuser');
    });
});
```

## Validation Queries
### SQL Assertions
```sql
-- Check for unique usernames
SELECT username, COUNT(*) FROM users GROUP BY username HAVING COUNT(*) > 1;

-- Check for valid foreign key references
SELECT * FROM users WHERE role_id IS NOT NULL AND role_id NOT IN (SELECT id FROM roles);

-- Check for non-null constraints
SELECT * FROM users WHERE username IS NULL OR password IS NULL;
```
