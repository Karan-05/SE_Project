## Problem Summary
Design a data pipeline for creating and managing learning spaces, posts, comments, and annotations in a ReactJS application. The pipeline will ensure data integrity and quality through validation checks and will document the schema, transformation steps, and testing strategies.
## Source Data and Schemas
- learning_spaces(id STRING PK, title STRING, description TEXT, prerequisites TEXT[], thumbnail STRING)
- posts(id STRING PK, space_id STRING FK, title STRING, body TEXT, created_at TIMESTAMP)
- comments(id STRING PK, post_id STRING FK, body TEXT, created_at TIMESTAMP)
- annotations(id STRING PK, post_id STRING FK, highlighted_text TEXT, user_annotation TEXT)
## Pipeline Skeleton (Python)
```python
function createLearningSpace(title, description, prerequisites, thumbnail) {
    // Validate inputs
    if (!title || !description) throw new Error('Title and description are required');
    // Save to database
    return db.collection('learning_spaces').add({ title, description, prerequisites, thumbnail });
}

function createPost(spaceId, title, body) {
    // Validate inputs
    if (!title || !body) throw new Error('Title and body are required');
    return db.collection('posts').add({ spaceId, title, body });
}
```
## Transformation Steps
Extract data from user inputs via React forms, transform data into structured formats for storage, and load into a database (e.g., Firebase or Supabase). The pipeline will also handle annotations and upvotes/downvotes for posts and comments.
## Test Strategy & Validation Queries
- Ensure no NULL values in required fields (title, description for learning spaces and posts).
- Validate that prerequisites are in a valid format (array of strings).
- Check for duplicate titles in learning spaces.
- Test: db.collection('learning_spaces').get().then(data => assert(data.length > 0));
- Test: db.collection('posts').get().then(data => assert(data.length > 0));
- Test: db.collection('comments').get().then(data => assert(data.length > 0));
