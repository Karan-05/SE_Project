# ETL/Data Pipeline Design for Topcoder Review App API Integration

## Problem Summary
The Topcoder Review App currently uses mock data for displaying active challenges on the My Active Challenge Page. The objective is to integrate real API functionality by replacing mock data with actual API calls to the Challenge API, ensuring that the frontend accurately reflects the current user's active challenges.

## Source Data and Schemas

### Source Schema (Challenge API)
The Challenge API returns data in the following structure:
```json
{
  "id": "12345",
  "name": "Challenge Name",
  "status": "ACTIVE",
  "currentPhaseEndDate": "2023-10-01T00:00:00Z",
  "memberId": "67890",
  "role": "participant",
  ...
}
```

### Destination Schema (ChallengeInfo Model)
The transformed data structure for the frontend:
```typescript
interface ChallengeInfo {
  id: string;
  name: string;
  status: string;
  currentPhaseEndDateString: string;
  timeLeft: string;
  timeLeftColor: string;
  timeLeftStatus: string;
  reviewProgress: number;
  role: string;
}
```

### Invariants
1. The `id` field must be unique for each challenge.
2. The `status` must always be "ACTIVE" for the challenges displayed.
3. The `currentPhaseEndDate` must be a valid date in the future.

## Transformation Steps
1. Fetch data from the Challenge API using the member ID.
2. Transform the API response to match the `ChallengeInfo` model.
3. Calculate derived fields such as `currentPhaseEndDateString`, `timeLeft`, `timeLeftColor`, `timeLeftStatus`, and `reviewProgress`.
4. Filter challenges based on the user's role.

## SQL or Code Snippets

### Pseudo-Code for API Integration
```typescript
async function fetchActiveReviews(memberId: string): Promise<ChallengeInfo[]> {
    const response = await fetch(`https://api.topcoder-dev.com/v6/challenges/?status=ACTIVE&memberId=${memberId}`);
    const data = await response.json();
    
    return data.map(challenge => transformChallenge(challenge));
}

function transformChallenge(challenge): ChallengeInfo {
    const timeLeft = calculateTimeLeft(challenge.currentPhaseEndDate);
    return {
        id: challenge.id,
        name: challenge.name,
        status: challenge.status,
        currentPhaseEndDateString: new Date(challenge.currentPhaseEndDate).toLocaleString(),
        timeLeft: timeLeft.remaining,
        timeLeftColor: timeLeft.color,
        timeLeftStatus: timeLeft.status,
        reviewProgress: calculateReviewProgress(challenge),
        role: challenge.role
    };
}
```

## Validation Queries
To ensure data integrity, the following SQL assertions can be run:
```sql
-- Check for unique IDs
SELECT id, COUNT(*) 
FROM challenges 
GROUP BY id 
HAVING COUNT(*) > 1;

-- Ensure status is always ACTIVE
SELECT COUNT(*) 
FROM challenges 
WHERE status != 'ACTIVE';

-- Validate future dates for currentPhaseEndDate
SELECT COUNT(*) 
FROM challenges 
WHERE currentPhaseEndDate <= NOW();
```
