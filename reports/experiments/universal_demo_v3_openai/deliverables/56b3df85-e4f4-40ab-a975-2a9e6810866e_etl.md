# Topcoder Review API - Challenge and Resource API Integration

## Problem Summary
The existing Review API (tc-review-api) does not correctly fetch pending reviews for users with different roles. The goal is to enhance the `GET /api/reviews` endpoint to integrate with the Challenge API and Resources API, ensuring that the endpoint returns properly formatted responses that include user role information and filters based on challenge status and user roles.

## Source Data and Schemas

### Review API Schema
```sql
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
    user_id INT,
    challenge_id INT,
    status VARCHAR(50),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### Challenge API Schema
```sql
CREATE TABLE challenges (
    id SERIAL PRIMARY KEY,
    status VARCHAR(50),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### Resources API Schema
```sql
CREATE TABLE resources (
    id SERIAL PRIMARY KEY,
    user_id INT,
    challenge_id INT,
    role VARCHAR(50),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### Invariants
1. Reviews must only be fetched if the associated challenge is active.
2. User roles must be correctly matched against the reviews.
3. The response must include user role information for each review.

## Transformation Steps
1. **Fetch Pending Reviews**: Retrieve reviews with status 'pending' from the Review API.
2. **Integrate Challenge API**: For each review, check if the associated challenge is active using the Challenge API.
3. **Integrate Resources API**: Fetch user roles for each challenge using the Resources API.
4. **Format Response**: Construct the response to include user role information and filter reviews based on challenge status and user roles.

## SQL or Code Snippets

### Pseudo-code for `getPendingReviews()`
```typescript
async getPendingReviews(userId: number) {
    const pendingReviews = await this.reviewService.getPendingReviews();
    
    const activeChallenges = await Promise.all(pendingReviews.map(review => 
        this.challengeApiService.getChallengeStatus(review.challenge_id)
    ));

    const userRoles = await Promise.all(activeChallenges.map(challenge => 
        this.resourceApiService.getUserRoles(challenge.id, userId)
    ));

    return this.formatResponse(pendingReviews, activeChallenges, userRoles);
}
```

## Validation Queries
```sql
-- Check if all fetched reviews are pending
SELECT COUNT(*) FROM reviews WHERE status = 'pending';

-- Ensure all challenges are active
SELECT COUNT(*) FROM challenges WHERE status = 'active';

-- Validate user roles are correctly associated with reviews
SELECT COUNT(*) FROM resources WHERE user_id = :userId AND challenge_id IN (SELECT challenge_id FROM reviews WHERE status = 'pending');
```
