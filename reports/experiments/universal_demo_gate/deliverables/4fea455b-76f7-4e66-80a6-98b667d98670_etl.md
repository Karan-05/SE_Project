## Problem Summary
Design an ETL pipeline to update the download function for challenge submitters, ensuring that only members who have submitted can download other submissions. The pipeline will include schema definitions, transformation steps, validation queries, and a checklist for acceptance.
## Source Data and Schemas
- submissions(user_id STRING, submission_id STRING, challenge_id STRING, status ENUM)
- user_settings(user_id STRING PK, can_download BOOLEAN)
## Pipeline Skeleton (Python)
```python
SELECT user_id, submission_id FROM submissions WHERE challenge_id = ? AND status = 'submitted';

UPDATE user_settings SET can_download = TRUE WHERE user_id IN (SELECT user_id FROM submissions WHERE challenge_id = ? AND status = 'submitted');
```
## Transformation Steps
Extract submission data from the database, transform it to check submission status, and load the updated permissions into the user settings table.
## Test Strategy & Validation Queries
- Ensure that all users who submitted have can_download set to TRUE.
- Check that no users who did not submit have can_download set to TRUE.
- Test: SELECT * FROM user_settings WHERE can_download = TRUE AND user_id NOT IN (SELECT user_id FROM submissions WHERE challenge_id = ? AND status = 'submitted');
- Test: SELECT * FROM user_settings WHERE can_download = FALSE AND user_id IN (SELECT user_id FROM submissions WHERE challenge_id = ? AND status = 'submitted');
