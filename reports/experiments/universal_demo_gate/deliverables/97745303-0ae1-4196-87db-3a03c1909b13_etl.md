## Problem Summary
Design an ETL pipeline to filter out winners from the submissions tab in the Online Review system, ensuring only valid submissions are displayed. The pipeline will include schema definitions, transformation steps, and validation queries.
## Source Data and Schemas
- submissions(submission_id STRING, user_id STRING, status ENUM, placement INT)
- filtered_submissions(submission_id STRING, user_id STRING, status ENUM, placement INT)
## Pipeline Skeleton (Python)
```python
SELECT * FROM submissions WHERE status != 'winner' AND placement NOT IN (4, 5);
```
## Transformation Steps
Extract data from the submissions database, filter out winners and invalid placements, transform the data for the submissions tab, and load the cleaned data back into the system.
## Test Strategy & Validation Queries
- Check that the count of filtered_submissions matches expected non-winner submissions.
- Ensure no entries in filtered_submissions have status 'winner'.
- Validate that placement values are within expected ranges.
- Test: SELECT COUNT(*) FROM filtered_submissions WHERE status = 'winner'; -- Should return 0
- Test: SELECT COUNT(*) FROM filtered_submissions WHERE placement IN (4, 5); -- Should return 0
