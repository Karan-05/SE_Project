We present an end-to-end audit of the Topcoder data pipeline and the workflow-control agents layered on top of it. By reprocessing the legacy Excel archive alongside refreshed API windows we recover 22,023 unique challenges ($26.5M in prizes, 5.9k AI-tagged tasks) and document every hop—from downloader scripts and JSON normalisation to MySQL schema control, research exports, and the new counterfactual branch-rollout dataset (`analysis/output/report.md`, `scripts/build_counterfactual_dataset.py`).

Using this reproducible stack we evaluate AEGIS, TARL, STRIDE, and the new counterfactual STRIDE (C-STRIDE) family. Hierarchical AEGIS still peaks at 62.5% success, TARL mimics the teacher without overriding, and legacy STRIDE residuals regress to ~0.78 success. Counterfactual gating is the only learned variant that avoids harm: the gate-only hybrid keeps 1.0 success while firing overrides on 8.9% of decisions with an observed win rate of 0.82 (`cstride_table_main.csv`). Every counterfactual value/residual head, however, collapses to 0.60–0.92 success with override regret ≈0.78 (`cstride_table_ablation.csv`) because the dataset only labels single-step branches. We also re-ran the real Topcoder SRM repos against the live OpenAI provider via `scripts/run_prompt_tuning_iteration.py` and the upgraded contract-aware harness: every learned strategy now hits localization precision 1.0 and implementation recall 1.0, yet contract coverage stalls at 0.38–0.67 and the semantic failure categories remain “aggregation” or “filtering,” confirming that the agents reach the right file but still violate the response contract.

The resulting paper is a hybrid/negative-result study: we release the ETL audit, research-ready tables, branch-rollout dataset, prompt-tuning workflow, and multi-seed C-STRIDE sweeps, and we argue that until counterfactual supervision spans entire override trajectories (and LLMs can patch multiple repo files coherently), the safest main method remains strong teacher imitation with calibrated, rarely triggered overrides.
