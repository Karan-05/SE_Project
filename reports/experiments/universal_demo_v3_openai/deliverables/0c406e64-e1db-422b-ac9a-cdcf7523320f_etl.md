# Topcoder Admin App - Consolidate Skills Manager

## Problem Summary
The goal of this project is to integrate the existing Skills Manager functionality into the Topcoder Admin App, specifically under a new "Platform" tab. This integration will simplify the management of skills for all administrators by removing the roles required for access and cleaning up unnecessary configurations.

## Source Data and Schemas

### Source Schema
The existing Skills Manager app has the following schema:

```sql
CREATE TABLE skills (
    skill_id SERIAL PRIMARY KEY,
    skill_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Destination Schema
The destination schema in the Admin App will be similar, with the addition of a new tab:

```sql
CREATE TABLE admin_skills (
    skill_id SERIAL PRIMARY KEY,
    skill_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Invariants
1. **Data Consistency**: The number of skills in the source must match the destination after the ETL process.
2. **Data Integrity**: No duplicate skills should exist in the destination.
3. **Timestamp Accuracy**: The `created_at` and `updated_at` fields should reflect the correct timestamps from the source.

## Transformation Steps
1. Extract data from the `skills` table.
2. Transform the data as necessary (e.g., renaming the table).
3. Load the transformed data into the `admin_skills` table.
4. Remove unnecessary configurations related to the old Skills Manager app.

## SQL or Code Snippets

```sql
-- Extract
SELECT * FROM skills;

-- Transform and Load
INSERT INTO admin_skills (skill_name, created_at, updated_at)
SELECT skill_name, created_at, updated_at FROM skills;
```

## Validation Queries
To ensure data integrity and correctness, the following SQL assertions will be run:

```sql
-- Check for data consistency
SELECT COUNT(*) FROM skills;
SELECT COUNT(*) FROM admin_skills;

-- Check for duplicate skills
SELECT skill_name, COUNT(*) 
FROM admin_skills 
GROUP BY skill_name 
HAVING COUNT(*) > 1;

-- Check timestamp accuracy
SELECT * FROM admin_skills WHERE created_at IS NULL OR updated_at IS NULL;
```
