# C-STRIDE Failure Analysis

- **Value heads over-ride everything** – The counterfactual value model learns large positive deltas for most states (beneficial fraction 0.55 in `results/aegis_rl/counterfactual/summary.json`), so gate+value variants fire in 32% of manager steps. Because we only evaluate the first macro of each branch, those overrides steer the simulator into unseen regions and tank success to 0.60 (`results/aegis_rl/cstride_cstride_gate_plus_value/cstride_gate_plus_value_summary.json`).
- **Residual bonuses cannot rescue value collapse** – Adding the residual policy or cost penalty simply reorders the same positive scores: `cstride_cost_aware` and `cstride_gate_plus_value_plus_residual` share identical metrics (0.60 success, override regret 0.78) because the residual bonus never overcomes the gate’s bias toward “always override.”
- **Feature ablations do not help** – Removing uncertainty or teacher confidence doesn’t fix the pathology. `cstride_without_uncertainty` inherits the same 0.60 success, while `cstride_without_teacher_confidence` improves to only 0.92 success with override win rates still <0.25.
- **Root cause** – Counterfactual branches evaluate only the first override action before handing control back to the teacher, so the dataset overestimates benefits of “try anything else.” Without multi-step counterfactuals, value heads can’t predict the downstream collapse and the gate lacks a notion of regret.
- **Mitigation / next steps** – Either (i) extend the dataset generator to roll out each alternate action until termination (so deltas reflect whole-episode outcomes), or (ii) teach the gate to reason about override sparsity by penalising overrides when predicted gains are <0. Higher thresholds alone do not work because the raw value scores remain positive; we need better counterfactual coverage.
