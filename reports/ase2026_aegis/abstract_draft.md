We present an end-to-end audit of the Topcoder data pipeline and the workflow-control agents layered on top of it. By reprocessing the legacy Excel archive alongside refreshed API windows we recover 22,023 unique challenges ($26.5M in prizes, 5.9k AI-tagged tasks), build an auditable 22k-index (`data/topcoder/corpus_index.jsonl`), filter to 3,966 runnable repos (`data/topcoder/executable_subset.jsonl`), and emit full-funnel snapshots (`data/topcoder/funnel_report.json`, `data/topcoder/source_acquisition_report.json`). Every URL discovered in the corpus is typed (`data/topcoder/artifact_candidates.jsonl`), filtered into repo candidates (`data/topcoder/repo_candidates.jsonl`), and either cloned/downloaded or rejected with explicit reasons (`data/topcoder/repo_fetch_manifest.jsonl`, `reports/ase2026_aegis/source_acquisition_snapshot.md`). We document every hop—from downloader scripts and JSON normalisation to MySQL schema control, research exports, the counterfactual branch-rollout dataset (`analysis/output/report.md`, `scripts/build_counterfactual_dataset.py`), and the new CGCS clause-graph archive (`src/decomposition/real_repo/contract_graph.py`, `scripts/build_cgcs_dataset.py`, `docs/CGCS_DATASET_SCHEMA.md`).

Using this reproducible stack we evaluate AEGIS, TARL, STRIDE, and the new counterfactual STRIDE (C-STRIDE) family. The latest AEGIS sweep (`reports/ase2026_aegis/table_main.csv`) still caps out at 50% success with negative reward, TARL mimics the teacher with 1.0 success and zero overrides, and the teacher-only STRIDE baseline now reaches 98.1% success with no overrides while residual heads fall back to 76% success and 0.24 harmful override rate (`reports/ase2026_aegis/stride_table_ablation.csv`). Counterfactual gating remains the only learned variant that avoids harm: the gate-only hybrid keeps 1.0 success while firing overrides on 8.9% of decisions with an observed win rate of 0.82 (`cstride_table_main.csv`). Every counterfactual value/residual head, however, collapses to 0.60–0.92 success with override regret ≈0.78 (`cstride_table_ablation.csv`) because the dataset only labels single-step branches. The CGCS build corroborates this: `data/cgcs/dataset_summary.json` reports 60 total rows and zero usable examples (missing clause IDs, weak contracts, or empty witnesses), and the funnel report shows no batch requests or solved tasks yet. We also re-ran the real Topcoder SRM repos against the live OpenAI provider via `scripts/run_prompt_tuning_iteration.py` and the upgraded contract-aware harness: every learned strategy now hits localization precision 1.0 and implementation recall 1.0, yet contract coverage stalls near 0.5 and the semantic failure categories remain “aggregation” or “filtering,” confirming that the agents reach the right file but still violate the response contract.

The resulting paper is a hybrid/negative-result study: we release the ETL audit, research-ready tables, branch-rollout dataset, CGCS clause-level logs, prompt-tuning workflow, and multi-seed C-STRIDE sweeps, and we argue that until counterfactual supervision spans entire override trajectories (and LLMs can patch multiple repo files coherently), the safest main method remains strong teacher imitation with calibrated, rarely triggered overrides.
