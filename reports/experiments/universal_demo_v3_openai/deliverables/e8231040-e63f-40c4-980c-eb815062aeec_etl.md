# Topcoder Online Review - ETL/Data Pipeline Design

## Problem Summary
The objective is to ensure that logged-out users are accurately represented in the UI navigation on the List Project page of the Topcoder Online Review platform. The current implementation may not reflect the logged-out state correctly, leading to confusion for users.

## Source Data and Schemas

### Source Data
- **User Table**: Contains user information including login status.
  
| Column Name | Data Type | Description               |
|-------------|-----------|---------------------------|
| user_id     | INT       | Unique identifier for user|
| username     | VARCHAR   | User's display name       |
| logged_in   | BOOLEAN   | User's login status       |

### Destination Schema
- **UI State Table**: Represents the state of the UI for users.

| Column Name | Data Type | Description                |
|-------------|-----------|----------------------------|
| user_id     | INT       | Unique identifier for user  |
| nav_state   | VARCHAR   | Navigation state (e.g., "logged_out", "logged_in") |

### Invariants
1. The `logged_in` field in the User Table must accurately reflect the `nav_state` in the UI State Table.
2. Each `user_id` in the UI State Table must exist in the User Table.
3. The `nav_state` must only contain valid states: "logged_out" or "logged_in".

## Transformation Steps
1. Extract user data from the User Table.
2. Transform the `logged_in` status into a corresponding `nav_state`:
   - If `logged_in` is `TRUE`, set `nav_state` to "logged_in".
   - If `logged_in` is `FALSE`, set `nav_state` to "logged_out".
3. Load the transformed data into the UI State Table.

## SQL or Code Snippets

```sql
-- SQL to transform and load data
INSERT INTO ui_state (user_id, nav_state)
SELECT user_id, 
       CASE 
           WHEN logged_in THEN 'logged_in' 
           ELSE 'logged_out' 
       END AS nav_state
FROM user;
```

## Validation Queries

```sql
-- Validation to check if all user_ids in UI State exist in User Table
SELECT COUNT(*) 
FROM ui_state 
WHERE user_id NOT IN (SELECT user_id FROM user);

-- Validation to ensure nav_state reflects logged_in status
SELECT COUNT(*) 
FROM ui_state 
WHERE nav_state = 'logged_in' AND user_id IN (SELECT user_id FROM user WHERE logged_in = FALSE);

SELECT COUNT(*) 
FROM ui_state 
WHERE nav_state = 'logged_out' AND user_id IN (SELECT user_id FROM user WHERE logged_in = TRUE);
```
