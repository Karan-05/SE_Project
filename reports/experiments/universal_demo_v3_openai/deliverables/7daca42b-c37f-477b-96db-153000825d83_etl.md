# Topgear Scorecard Issue ETL/Data Pipeline Design

## Problem Summary
The Topgear Scorecard application is experiencing inconsistencies in dropdown options compared to the previous version. This ETL pipeline aims to extract, transform, and load the correct scorecard data to ensure that the dropdown options are consistent with the old application. The goal is to update the SQL file to reflect the correct data for the Topgear Scorecard Issue while maintaining existing functionalities.

## Source Data and Schemas
### Source Schema
- **Table: `scorecard_options`**
  - `id` (INT, PK)
  - `option_name` (VARCHAR)
  - `category` (VARCHAR)
  - `created_at` (DATETIME)

### Destination Schema
- **Table: `updated_scorecard_options`**
  - `id` (INT, PK)
  - `option_name` (VARCHAR)
  - `category` (VARCHAR)
  - `updated_at` (DATETIME)

### Invariants
1. The `id` in `updated_scorecard_options` must match the `id` in `scorecard_options`.
2. The `option_name` must be unique within the same `category`.
3. The `created_at` timestamp should not be altered during the transformation.

## Transformation Steps
1. **Extract** the current data from `scorecard_options`.
2. **Transform** the data to ensure dropdown options are consistent with the old application:
   - Filter out any deprecated options.
   - Update option names if necessary.
3. **Load** the transformed data into `updated_scorecard_options`.

## SQL or Code Snippets
```sql
-- Step 1: Extract
WITH extracted_options AS (
    SELECT id, option_name, category, created_at
    FROM scorecard_options
)

-- Step 2: Transform
INSERT INTO updated_scorecard_options (id, option_name, category, updated_at)
SELECT 
    id, 
    option_name, 
    category, 
    NOW() AS updated_at
FROM extracted_options
WHERE option_name IN (SELECT option_name FROM old_app_options);
```

## Validation Queries
```sql
-- Check for matching IDs
SELECT COUNT(*) FROM updated_scorecard_options 
WHERE id NOT IN (SELECT id FROM scorecard_options);

-- Validate unique option names within categories
SELECT category, option_name, COUNT(*)
FROM updated_scorecard_options
GROUP BY category, option_name
HAVING COUNT(*) > 1;

-- Ensure created_at timestamps are unchanged
SELECT COUNT(*) 
FROM updated_scorecard_options 
WHERE created_at IS NULL;
```
