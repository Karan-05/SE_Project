# ETL/Data Pipeline Design for Topcoder O/R F2F - Submissions Tab

## Problem Summary
The goal is to remove the display of winners and additional placements from the Submissions/Screening tab in the Topcoder platform. Currently, the winners and extra placements are incorrectly shown in this tab, which should only display non-winning submissions.

## Source Data and Schemas
### Source Schema
- **Submissions Table**
  - `submission_id` (INT, PK)
  - `challenge_id` (INT)
  - `user_id` (INT)
  - `status` (VARCHAR)
  - `placement` (INT)

### Destination Schema
- **Filtered Submissions Table**
  - `submission_id` (INT, PK)
  - `challenge_id` (INT)
  - `user_id` (INT)
  - `status` (VARCHAR)

### Invariants
1. The number of submissions in the Filtered Submissions Table should be less than or equal to that in the original Submissions Table.
2. All entries in the Filtered Submissions Table must have a `placement` value greater than 3.
3. The `status` field in the Filtered Submissions Table must remain unchanged from the original submissions.

## Transformation Steps
1. Extract data from the Submissions Table.
2. Filter out submissions where `placement` is 1, 2, or 3 (winners).
3. Load the filtered data into the Filtered Submissions Table.

## SQL or Code Snippets
```sql
-- SQL to filter out winners and load into a new table
CREATE TABLE Filtered_Submissions AS
SELECT submission_id, challenge_id, user_id, status
FROM Submissions
WHERE placement > 3;
```

```python
# Pseudo-code for the ETL process
def etl_process():
    # Extract
    submissions = extract_data("Submissions")
    
    # Transform
    filtered_submissions = [s for s in submissions if s['placement'] > 3]
    
    # Load
    load_data("Filtered_Submissions", filtered_submissions)
```

## Validation Queries
```sql
-- Ensure no winners are present
SELECT COUNT(*) FROM Filtered_Submissions WHERE placement <= 3;

-- Verify that the number of submissions is less than or equal to the original
SELECT COUNT(*) FROM Filtered_Submissions;
SELECT COUNT(*) FROM Submissions;

-- Check that status remains unchanged
SELECT DISTINCT status FROM Submissions
EXCEPT
SELECT DISTINCT status FROM Filtered_Submissions;
```
