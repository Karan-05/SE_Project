# ETL/Data Pipeline Design for FICO Falcon Engineer Role

## Problem Summary
The goal of this ETL/data pipeline is to efficiently deploy, configure, and maintain the FICO Falcon Fraud Manager suite in AWS Cloud environments. The pipeline will handle data ingestion from various sources, perform necessary transformations, and load the data into the Falcon system for fraud detection and management. This will ensure that the system is optimized for performance and scalability while maintaining data integrity.

## Source Data and Schemas
### Source Data
1. **Transaction Data**
   - **Schema**: 
     - `transaction_id` (UUID)
     - `amount` (Decimal)
     - `timestamp` (Timestamp)
     - `user_id` (UUID)
     - `merchant_id` (UUID)

2. **User Profile Data**
   - **Schema**: 
     - `user_id` (UUID)
     - `name` (String)
     - `email` (String)
     - `account_creation_date` (Date)

### Destination Schema
- **Falcon Data Schema**
  - `falcon_id` (UUID)
  - `transaction_amount` (Decimal)
  - `transaction_time` (Timestamp)
  - `user_identifier` (UUID)
  - `merchant_identifier` (UUID)
  - `risk_score` (Integer)

### Invariants
1. **Uniqueness**: Each `transaction_id` must be unique.
2. **Non-null Constraints**: `user_id` and `amount` cannot be null in the transaction data.
3. **Data Type Integrity**: All fields must adhere to their defined data types.

## Transformation Steps
1. **Data Cleaning**: Remove any duplicate or null values from the source datasets.
2. **Data Enrichment**: Join transaction data with user profile data to add user details to the transactions.
3. **Risk Scoring**: Implement a risk scoring algorithm based on transaction amount and user profile attributes.
4. **Data Formatting**: Map source fields to the destination schema.

## SQL or Code Snippets
```sql
-- Data Cleaning
WITH cleaned_transactions AS (
    SELECT DISTINCT transaction_id, amount, timestamp, user_id, merchant_id
    FROM transactions
    WHERE user_id IS NOT NULL AND amount IS NOT NULL
),

-- Data Enrichment
enriched_data AS (
    SELECT ct.transaction_id, ct.amount, ct.timestamp, ct.user_id, ct.merchant_id, up.name, up.email
    FROM cleaned_transactions ct
    JOIN user_profiles up ON ct.user_id = up.user_id
)

-- Final Insert into Falcon Schema
INSERT INTO falcon_data (falcon_id, transaction_amount, transaction_time, user_identifier, merchant_identifier, risk_score)
SELECT 
    transaction_id, 
    amount, 
    timestamp, 
    user_id, 
    merchant_id, 
    CASE 
        WHEN amount > 1000 THEN 5 
        ELSE 1 
    END AS risk_score
FROM enriched_data;
```
