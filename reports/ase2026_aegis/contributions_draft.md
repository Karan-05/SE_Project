1. **Reproducible Topcoder ETL audit** – We regenerate the legacy Excel archive (`challenge_data/challengeData_legacy_xlsx/page1.json`) and re-run every reporting/export script so reviewers can rebuild the 22,023-challenge corpus end-to-end (`analysis/output/report.md`, `data/raw`, `data/processed`) **plus** the new manifest files: `data/topcoder/corpus_index.jsonl`, `data/topcoder/executable_subset.jsonl`, `data/topcoder/artifact_candidates.jsonl`, `data/topcoder/repo_candidates.jsonl`, `data/topcoder/repo_fetch_manifest.jsonl`, `data/topcoder/source_acquisition_report.json`, and `data/topcoder/funnel_report.json` that quantify attrition at each stage (16,562 likely executable repos, 3,966 runnable challenges, 60 CGCS rows, 9 eval items, 0 batch successes).
2. **Counterfactual override dataset** – We introduce `scripts/build_counterfactual_dataset.py`, which clones the simulator at each teacher decision, replays alternate macros, and produces 4,925 calibrated decision states + 10,144 branch rollouts (`results/aegis_rl/counterfactual/*`) for offline gate/value experiments.
3. **Honest assessment of learned overrides** – Multi-seed runs of AEGIS, TARL, STRIDE, and the new C-STRIDE variants (`reports/ase2026_aegis/{table_main,cstride_table_main,cstride_table_ablation}.csv`) show that the STRIDE teacher baseline holds at 98.1% success with zero overrides, residual heads crash to 76% success with 0.246 regret, and only the counterfactual gate-only hybrid preserves teacher performance (8.9% overrides, win rate 0.82). The flagship method is still teacher imitation, and the paper frames the counterfactual analysis as a diagnostic, not a superior controller.
4. **CGCS clause-graph instrumentation + dataset** – `src/decomposition/real_repo/{witnesses,contract_graph}.py` capture contract clauses, regression guards, and witnesses for every repo attempt; `scripts/build_cgcs_dataset.py` and `docs/CGCS_DATASET_SCHEMA.md` release the clause-level dataset (`data/cgcs/{train,dev,test}.jsonl`), and `scripts/make_paper_{tables,figures}.py` summarize the discharge metrics in `reports/ase2026_aegis`.
5. **Real-repo diagnostic harness** – `scripts/run_prompt_tuning_iteration.py` plus the upgraded repo runner (`src/decomposition/real_repo/harness.py`, `run_real_repo_benchmark.py`) now add contract-aware prompts, semantic checklists/critics, snapshot validation, and contract-coverage metrics. Running the SRM backlog tasks against OpenAI gpt‑4.1‑mini shows localization precision = 1.0 yet contract coverage ≈0.5 with dominant aggregation/filtering failures, proving the remaining gap is semantic contract satisfaction rather than file targeting or plumbing.
