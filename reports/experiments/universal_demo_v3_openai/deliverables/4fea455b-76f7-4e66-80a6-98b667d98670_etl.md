# ETL/Data Pipeline Design for Submission Download Functionality

## Problem Summary
The goal is to update the download functionality for challenge submitters in the Topcoder platform. Currently, members cannot download submissions unless they have submitted to the challenge. This update will allow members who have submitted to download other submissions, serving as a learning opportunity.

## Source Data and Schemas
### Source Schema
**Submissions Table**
| Column Name      | Data Type   | Description                              |
|------------------|-------------|------------------------------------------|
| submission_id    | INT         | Unique identifier for each submission    |
| member_id        | INT         | Unique identifier for each member        |
| challenge_id     | INT         | Unique identifier for each challenge     |
| submission_date   | TIMESTAMP   | Date and time of submission              |
| content          | TEXT        | The content of the submission            |

### Destination Schema
**Download Permissions Table**
| Column Name      | Data Type   | Description                              |
|------------------|-------------|------------------------------------------|
| member_id        | INT         | Unique identifier for each member        |
| can_download     | BOOLEAN     | Indicates if the member can download     |

## Transformation Steps
1. **Extract**: Pull data from the `Submissions` table to identify which members have submitted to challenges.
2. **Transform**: Create a new table `Download Permissions` that indicates whether a member can download submissions based on their submission history.
3. **Load**: Insert the transformed data into the `Download Permissions` table.

## SQL or Code Snippets
```sql
-- Extracting data from Submissions table
WITH MemberSubmissions AS (
    SELECT DISTINCT member_id
    FROM Submissions
)

-- Transforming and loading data into Download Permissions table
INSERT INTO Download_Permissions (member_id, can_download)
SELECT member_id, TRUE
FROM MemberSubmissions;

-- Setting can_download to FALSE for members without submissions
UPDATE Download_Permissions
SET can_download = FALSE
WHERE member_id NOT IN (SELECT DISTINCT member_id FROM MemberSubmissions);
```

## Validation Queries
```sql
-- Check if all members who submitted can download
SELECT COUNT(DISTINCT member_id) AS members_with_download_permission
FROM Download_Permissions
WHERE can_download = TRUE;

-- Ensure no member without a submission can download
SELECT COUNT(DISTINCT member_id) AS members_without_submission
FROM Download_Permissions
WHERE can_download = TRUE AND member_id NOT IN (SELECT DISTINCT member_id FROM Submissions);
```
