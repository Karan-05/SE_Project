## Requirements
- Integrate Challenge API into My Active Challenge Page
- Replace mock data with real API calls
- Ensure proper error handling and loading states

## Components
- **ActiveReviewsPage**: Fetches and displays active challenges.
- **reviews.service.ts**: Handles API calls.
- **useFetchActiveReviews.ts**: Custom hook for fetching data.
- **ChallengeInfo**: Data model for challenges.

## Interfaces
- **ChallengeInfo**: Defines the structure of challenge data.
- **AuthService**: Manages authentication and token retrieval.

## Risks
- API rate limits may affect performance.
- Handling of empty data scenarios could lead to poor user experience.

## Rollout Plan
1. Implement API integration in a feature branch.
2. Conduct thorough testing of API calls and data handling.
3. Merge to main branch after validation.
4. Monitor for issues post-deployment.
